云计算技术实验报告

利用云平台搭建个人博客

一、实验目的

本实验旨在通过实践操作，掌握基于云计算环境的网站部署技能。具体包括：

  • 熟悉 GitHub Pages 云平台的基本使用方法
  • 掌握静态网站的开发和部署流程
  • 实现个人博客的搭建和内容发布
  • 将实验报告以博文形式发布在个人博客上

二、实验环境

硬件环境

  • 联网的计算机一台（处理器：Intel Core i5，内存：8GB）

软件环境

  • 操作系统：Windows 11
  • 浏览器：Chrome / Edge
  • 版本控制：Git
  • 云平台：GitHub Pages

三、实验步骤

1. 创建 GitHub 仓库

登录 GitHub 账号，创建新仓库，仓库名称为 [用户名].github.io，设置为 Public 可见性。

2. 开发博客首页

创建 index.html 作为博客首页，包含博客标题、导航栏、文章列表等基本元素。

<!DOCTYPE html>
<html>
<head>
    <title>我的个人博客</title>
</head>
<body>
    <h1>欢迎来到我的博客</h1>
    <p>这是我利用GitHub Pages搭建的个人博客。</p>
</body>
</html>

3. 创建实验报告博文

创建 posts/实验报告.html 文件，将本实验报告内容作为一篇博文发布。

4. 部署到 GitHub Pages

提交代码到 GitHub 仓库，并在 Settings → Pages 中配置部署分支为 main。

# 提交代码
git add .
git commit -m "发布个人博客"
git push origin main

5. 验证部署结果

等待 1-3 分钟后，访问 https://[用户名].github.io 查看博客。

博客访问地址

博客链接： https://[你的用户名].github.io

实验报告博文链接： https://[你的用户名].github.io/posts/实验报告.html

五、结论分析与体会

5.1 实验结论

通过本次实验，成功完成了以下目标：

  • 在 GitHub 云平台上成功搭建个人博客网站
  • 掌握了 GitHub Pages 静态网站托管服务的使用方法
  • 实现了个人博客的部署、访问和内容更新
  • 将实验报告以博文形式发布在个人博客上

5.2 GitHub Pages 云平台的优势

  • 免费：无需购买服务器和域名，零成本搭建个人网站
  • 简单：只需创建仓库、上传文件即可完成部署
  • 版本控制：基于 Git 管理，所有修改都有记录
  • 自动部署：代码推送后自动触发部署，无需手动操作
  • 稳定可靠：依托 GitHub 全球 CDN 网络，访问速度快

5.3 遇到的问题及解决方法

问题 解决方法
部署后页面无法访问 检查仓库名称是否为 用户名.github.io 格式
修改后页面未更新 等待 1-3 分钟部署完成，按 Ctrl+F5 强制刷新
页面显示乱码 添加 <meta charset="UTF-8"> 标签

5.4 实验体会

通过本次实验，我深刻体会到了云计算技术带来的便利。GitHub Pages 作为云平台服务，让我无需关心服务器配置、环境搭建等复杂问题，只需要专注于内容创作本身。这种"拿来即用"的云服务模式，大大降低了网站部署的门槛。

同时，我也理解了静态网站托管的基本原理，掌握了 Git 版本控制工具的基本使用。这些技能不仅适用于本次实验，在未来的学习和工作中也会有广泛应用。

云计算技术的核心价值在于将基础设施抽象化、服务化，让开发者能够更专注于业务逻辑。本次实验很好地体现了这一理念。

5.5 未来展望

本次实验为我打开了云计算应用的大门，后续可以进一步探索：

  • 使用 Hexo、Jekyll 等静态博客框架，提升博客管理效率
  • 购买自定义域名，绑定到 GitHub Pages
  • 学习其他云平台（如阿里云、腾讯云）的动态网站部署
  • 探索云数据库、云存储等更多云服务
